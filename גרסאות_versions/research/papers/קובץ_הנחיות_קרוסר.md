# 📘 קובץ הנחיות לקרוסר: שילוב תמונות ממודלים והכשרת מערכת לזיהוי אתרוגים

## 🎯 מטרה

לבנות מערכת שמזהה אוטומטית אתרוגים לפי כשרותם (מהודר / כשר / פסול) וסוג הפגם או הזן – בהתבסס על מודלים תלת־ממדיים ותמונות מכל זווית. מטרת הקובץ: להסביר לצוות הקרוס (Crowdsource) איך לאסוף, לסווג ולשלב את התמונות בצורה מדויקת כך שהמערכת תלמד לזהות נכון כל אתרוג לפי הקריטריונים ההלכתיים.

## 🧩 שלב 1: איסוף תמונות ליצירת מודלים

### 📸 פרוטוקול צילום מדויק

#### **ציוד נדרש:**
- מצלמה איכותית (≥12MP)
- פלטפורמה מסתובבת 360°
- תאורה אחידה (2 מקורות אור)
- רקע לבן/שחור חלק
- סרגל מדידה לסקאלה

#### **זוויות צילום חובה:**
1. **360° מהצד** – פלטה מסתובבת, צילום כל 15° (24 תמונות)
2. **זווית עליונה (45°)** – סיבוב נוסף, מזווית שמצלמת את הפיטם
3. **צילום עליון (Top View)** – תקריב ישיר על הפיטם
4. **בסיס האתרוג (Bottom View)** – צילום תחתון לאחר הפיכה

#### **⚠️ מניעת זיהוי פנים:**
- **אין לצלם אנשים** - רק את האתרוג
- **השתמש בטיימר** או שלט רחוק
- **בדוק תמונה לפני שמירה** - וודא שאין פנים ברקע
- **אם יש פנים בתמונה** - מחק וצלם מחדש

### 🎯 דרישות איכות:
- תאורה אחידה ללא צללים חדשים
- רקע נקי ללא חפצים או אנשים
- אתרוג נקי ללא אבק או לכלוך
- פוקוס חד על כל התמונות
- רזולוציה מינימלית: 12MP

## 🤖 שלב 4: הכשרת המערכת על בסיס המאגרים

### 🔧 עיבוד תלת-ממדי:

#### **תוכנות מומלצות:**
- **Meshroom** (חינמי) - איכות גבוהה, עיבוד איטי
- **RealityCapture** (בתשלום) - מהיר ומדויק
- **Metashape** (בתשלום) - מקצועי ויציב

#### **פרוטוקול עיבוד:**
1. **ייבוא תמונות** - העלה את כל התמונות (24-30 תמונות)
2. **Feature Matching** - זיהוי נקודות משותפות
3. **Structure from Motion** - בניית נקודות תלת-ממד
4. **Dense Reconstruction** - יצירת mesh צפוף
5. **Texturing** - הוספת טקסטורה לתמונות
6. **Export** - ייצוא ל-obj/fbx/ply

## 🔍 שלב 5: חיבור לתהליך למידת מכונה

### 📊 דרישות מינימליות:
- **500-1000 אתרוגים** מתויגים
- **תמונות איכותיות** מכל זווית
- **תיוג מדויק** ועקבי
- **אין פנים** באף תמונה

### 🧠 סוגי מודלים:

#### **1. מודל סיווג (Classification):**
- קלט: תמונה אחת
- פלט: כשרות (מהודר/כשר/פסול)
- דיוק צפוי: 85-95%

#### **2. מודל זיהוי (Detection):**
- קלט: תמונה אחת
- פלט: מיקום וסוג פגם
- דיוק צפוי: 80-90%

#### **3. מודל תלת-ממדי (3D Analysis):**
- קלט: מודל תלת-ממדי + תמונות
- פלט: ניתוח טקסטורה ובליטות
- דיוק צפוי: 90-98%

### 🔧 ארכיטקטורת המערכת:

```python
# דוגמה לארכיטקטורה
class EtrogAnalyzer:
    def __init__(self):
        self.classification_model = load_classification_model()
        self.detection_model = load_detection_model()
        self.face_detector = load_face_detector()
    
    def analyze_etrog(self, images):
        # בדיקת פנים
        if self.contains_faces(images):
            return {"error": "Faces detected in images"}
        
        # ניתוח תמונות
        results = []
        for image in images:
            kosher_status = self.classification_model.predict(image)
            defects = self.detection_model.detect(image)
            results.append({
                "kosher_status": kosher_status,
                "defects": defects
            })
        
        return self.combine_results(results)
```

## ✅ לסיכום

### 🎯 המטרה:
כל אתרוג יהפוך לסט נתונים שלם → תמונות → מודל תלת-ממד → תיוג → אימון → זיהוי אוטומטי.

### 📋 שלבי העבודה:
1. **צילום מדויק** - 24 זוויות + זוויות משלימות
2. **בדיקת איכות** - אין פנים, תאורה אחידה
3. **תיוג מקצועי** - כשרות, פגמים, זן
4. **עיבוד תלת-ממדי** - יצירת מודל איכותי
5. **אימון מודל** - למידת מכונה
6. **זיהוי אוטומטי** - תוצאות מדויקות

### 🚀 תוצאות צפויות:
- **דיוק זיהוי**: 90-95%
- **מהירות ניתוח**: <30 שניות לאתרוג
- **אוטומציה מלאה**: ללא התערבות ידנית
- **תיעוד מקצועי**: דוחות מפורטים

אם תעבדו לפי שלבים אלה, תוך הקפדה על זוויות, תיוג ופירוט – המערכת תוכל ללמוד לזהות כל אתרוג לפי מאפיינים הלכתיים, ברמת דיוק גבוהה ואובייקטיבית.
